# Law Zero：Claude Code 自我进化框架

> **你的 Claude Code 昨天学的东西今天就忘了。Law Zero 让它记住——而且每次升级都变得更聪明。**

---

## 问题

你花了几个小时教 Claude Code 你项目的规则、工具和流程。然后你升级了一个脚本、添加了一个新技能、改了一个配置——突然：

- 旧指令和新指令打架
- 已废弃的方法还在被推荐
- 同样的错误再次发生，因为教训丢了
- `CLAUDE.md` 说一套，`SKILL.md` 说另一套

**Claude Code 没有内置机制来在变更后保持自身一致性。**

## 解决方案：Law Zero（第零法则）

Law Zero 是 Claude Code 的**自我进化框架**。它是一条凌驾于所有其他规则之上的"第零法则"，确保你的 AI 助手在任何升级后自动维护全局一致性。

任何东西升级时，Law Zero 触发 4 步协议：

```
升级 → 删除旧版 → 全局扫描冲突 → 验证汇报
```

就这么简单。没有复杂配置，不需要外部工具。把它放到 `CLAUDE.md` 顶部，你的 Claude Code 就能自我维护了。

---

## 快速开始

把以下内容加到你的 `CLAUDE.md` **最顶部**：

```markdown
## 第零法则：自我进化机制（最高优先级！所有规则之上！）

任何升级操作触发时，立即执行 4 步协议：

1. **升级实施** — 完成新版本部署
2. **删除旧版** — 彻底清除所有被替代的旧内容（代码、配置、文档、注释、教训条目）
3. **全局一致性扫描** — 扫描所有相关文件，发现矛盾/冲突/过时内容立即修正
4. **验证汇报** — 确认一致性，汇报改了什么、删了什么、修了什么

扫描范围：CLAUDE.md、memory文件、lessons文件、SKILL.md
绝不允许新旧并存。发现矛盾直接修正，不需询问。
```

详细模板和教训学习系统见 [`templates/CLAUDE.md`](templates/CLAUDE.md)。

## 真实案例

这个框架诞生于真实的 Claude Code 远程控制场景（通过飞书控制 Mac）。经过数周日常使用发现：

- 8 个配置文件中有 **15 处矛盾指令**
- 技能文档还在推荐**已废弃的方法**
- 教训库和 `CLAUDE.md` 教的东西相反
- 新升级默默破坏了旧工作流

实施 Law Zero 后，每次升级自动触发扫描，一次性发现并修复了所有 15 处冲突。
